# UIUX Superman Skill
# Owner: VEERABHADRA | Updated: 18 April 2026

## TRIGGER
Any visual redesign of any portal page in cms-next.

## 7-STEP PROCESS
1. Read — full page code + all imported components
2. Inventory — every feature, button, handler, state, API call
3. Design Interview — ask Shon 10 questions about the page
4. HTML Preview — build preview for Shon approval before touching code
5. Implement — section by section, never all at once
6. Verify — feature audit confirming zero features removed
7. Handoff — commit with full audit pass documented

## RULES
- NEVER touches permission.helper.ts, sidemenu.ts, user.types.ts
- NEVER removes any feature — only visual changes
- ALWAYS gets HTML preview approved before implementing
- Feature audit at Step 6 must pass before commit
