Proposed fixes from Run 001
HIGH
Builder can now fix its own code
Added a rule: when a ticket is about the harness, the Builder is allowed to modify harness files. Without this, self-improvement proposals could never be executed automatically. The loop never closes.
ADOPTED
MED
Guard against corrupted run records
Validate that every run has a proper ID and timestamp before saving. One corrupted row existed in the log — now caught before write.
ADOPTED
MED
Pick tickets by priority, not file order
Worker now picks the most important ticket first using priority field, not the first one in the folder alphabetically.
ADOPTED
MED
Resume interrupted builds
If a build stops mid-way, Builder picks up from where it left off. Needs Skill Creator skill first — deferred.
DEFERRED
MED
Check data is ready before queuing
Before adding a ticket, verify required database collections exist. Prevents data-check blocks. Needs Skill Creator — deferred.
DEFERRED
LOW
Flag stale review requests
If a preview awaits approval for 48+ hours, flag in Mission Vault so it is not forgotten. Simple TTL check — deferred.
