What this capability is

◎
A meta-harness that reads how the brain has been working and proposes how it can work better
Reads five SOP files, the eagle-harness JSONL log, the ticket queue, and a high-level codebase summary. Asks one structured LLM call: what patterns are emerging? what's recurring? what should the brain learn? Writes a markdown report to intelligence/proposed-fixes/, appends one new dated section to LEARNING-MIND.md, and logs one JSONL row with the same schema as the eagle harness. Mission Control's existing Self-Improvement — Intelligence Reports panel picks it up automatically.

Trigger surface — manual only for v1

# run a normal analysis
$ node ~/deassists-workspace/369-brain/harness/self-improvement/self-improvement-harness.js

# dry-run — assemble the prompt, print it, do not call Claude
$ node …/self-improvement-harness.js --dry-run

# skip the LEARNING-MIND append
$ node …/self-improvement-harness.js --skip-learning-mind

# consider only the last N harness-runs entries (default: all)
$ node …/self-improvement-harness.js --limit-runs 20

# exit codes
0 complete   1 failed mid-run   2 blocked (a Tier-1 SOP file missing)

Edge cases (brainstormed)

  • Empty harness-runs log — analyzer must produce a valid report with "no patterns yet, baseline established" note. Tested in __tests__/inputs.test.js.
  • Tier-1 SOP file missing — exit code 2, no report written, no LEARNING-MIND append, JSONL row records status: "blocked".
  • Claude binary not on PATH — exit code 1 at invoke-analyzer phase, JSONL row records status: "failed" with the binary path attempted.
  • Disk full mid-write — partial files cleaned up in a finally block; LEARNING-MIND append uses temp-file-then-rename pattern.
  • Concurrent invocations — second invocation aborts at workspace lock acquisition (uses same workspace.js primitive eagle-harness uses).
  • LEARNING-MIND.md does not yet have a closing newline — writer prepends \n\n to ensure clean section separation.
  • --dry-run combined with --skip-learning-mind — both honored; no file writes anywhere; prompt printed to stdout only.
